# 阿里云 FC 3.0 详细部署指南

## 🎯 当前任务：创建 Web 函数

---

## 第一步：进入正确的页面

### 1.1 在函数计算控制台首页
你会看到两个大选项：
- **"函数"** ← 选择这个！
- "应用"

点击左侧的 **"函数"** 菜单。

### 1.2 点击"创建函数"按钮
在右上角，有一个蓝色的 **"创建函数"** 按钮，点击它。

---

## 第二步：选择函数类型

### 2.1 你会看到多个选项
FC 3.0 的界面会显示：
- **Web 函数** ← 选择这个！
- 事件函数
- 任务函数
- GPU 函数

**选择"Web 函数"**，因为你的后端是 API 服务。

---

## 第三步：配置函数（重要！）

### 3.1 基本配置页面

#### ① 函数名称
```
tianba-kitchen-api
```

#### ② 运行环境
选择：**Node.js 18** 或 **自定义运行时**

**重要**：如果要上传代码包，选择"Node.js 18"

---

### 3.2 代码配置（这里是上传入口！）

#### 方式一：使用代码包上传（推荐）

1. 找到 **"代码配置"** 或 **"部署方式"** 区域
2. 选择 **"通过代码包上传"**
3. 点击 **"选择文件"** 或 **"上传代码包"** 按钮
4. 选择我准备好的文件：`tianba-backend-deploy.tar.gz`

#### 方式二：使用在线编辑

1. 选择 **"在线编辑"**
2. 把代码复制粘贴进去

---

### 3.3 高级配置（展开查看）

#### ① 内存规格
选择：**512 MB**

#### ② 执行超时时间
输入：**60** 秒

#### ③ 监听端口
输入：**9000**

#### ④ 环境变量（重要！）
点击"环境变量"或"高级配置"，添加：

```
SUPABASE_URL = 你的Supabase项目URL
SUPABASE_ANON_KEY = 你的Supabase密钥
COZE_API_KEY = 你的Coze API密钥
NODE_ENV = production
PORT = 9000
```

---

## 第四步：创建触发器（让外部能访问）

### 4.1 在创建函数页面
找到 **"触发器配置"** 区域

### 4.2 创建 HTTP 触发器
1. 点击 **"创建触发器"**
2. 填写：
   - 触发器类型：**HTTP 触发器**
   - 触发器名称：`http-trigger`
   - 认证方式：**anonymous**（匿名访问）
   - 请求方式：勾选全部（GET、POST、PUT、DELETE、PATCH、HEAD、OPTIONS）

---

## 第五步：点击创建

检查所有配置后，点击底部的 **"创建"** 按钮。

---
