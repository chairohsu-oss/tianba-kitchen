# 🌟 超级小白部署指南（不需要命令行）

## 第一步：下载代码包

在 Coze 平台的文件管理器中，下载这个文件：
- `tianba-server.tar.gz`（项目根目录）

---

## 第二步：创建函数计算服务（全部网页操作）

### 2.1 进入函数计算控制台
1. 在阿里云首页搜索框输入"函数计算"
2. 点击搜索结果进入控制台
3. 如果提示开通服务，点击"立即开通"（免费）

### 2.2 创建服务
1. 点击左侧菜单"服务及函数"
2. 点击右上角蓝色按钮"创建服务"
3. 填写：
   - 服务名称：`tianba-kitchen-service`
   - 描述：天霸私厨后端
4. 点击"确定"

---

## 第三步：创建函数

### 3.1 开始创建
1. 点击刚创建的服务名称进入
2. 点击"创建函数"按钮
3. 选择"事件函数"（重要！）
4. 点击"下一步"

### 3.2 填写基本信息
- 函数名称：`tianba-kitchen-api`
- 运行环境：选择 **Node.js 18**
- 请求处理程序：`dist/main.handler`

### 3.3 上传代码
1. 代码上传方式：选择"通过 ZIP 包上传"
2. 点击"选择文件"
3. 选择刚才下载的 `tianba-server.tar.gz`
4. 等待上传完成

### 3.4 配置规格
- 内存规格：**512 MB**
- vCPU：0.35 核
- 执行超时时间：**60 秒**
- 单实例并发度：**10**

### 3.5 配置环境变量（重要！）

点击"高级配置" → "环境变量"，添加以下变量：

| 变量名 | 变量值 | 说明 |
|--------|--------|------|
| SUPABASE_URL | 你的Supabase项目URL | 从 Supabase 控制台获取 |
| SUPABASE_ANON_KEY | 你的Supabase密钥 | 从 Supabase 控制台获取 |
| COZE_API_KEY | 你的Coze API密钥 | 从 Coze 平台获取 |
| NODE_ENV | production | 生产环境标识 |
| PORT | 9000 | 服务端口 |

**获取方式**：
- `SUPABASE_URL` 和 `SUPABASE_ANON_KEY`：
  1. 登录 Supabase 控制台
  2. 进入你的项目
  3. 点击左侧"设置" → "API"
  4. 复制 URL 和 anon public key

- `COZE_API_KEY`：
  1. 登录 Coze 平台
  2. 点击右上角头像 → "API Key"
  3. 复制你的 API Key

### 3.6 配置触发器（让外部能访问）

1. 点击"触发器配置"
2. 点击"创建触发器"
3. 填写：
   - 触发器类型：**HTTP 触发器**
   - 触发器名称：`http-trigger`
   - 认证方式：**anonymous**（匿名访问）
   - 请求方式：勾选全部（GET、POST、PUT、DELETE、PATCH、HEAD、OPTIONS）

### 3.7 点击创建
1. 检查所有配置
2. 点击底部蓝色按钮"创建"

---

## 第四步：获取 API 地址

创建成功后（需要等待 1-2 分钟）：

1. 在函数详情页，找到"触发器管理"
2. 你会看到一个"公网访问地址"，类似：
   ```
   https://xxxxxx.cn-shanghai.fc.aliyuncs.com/2016-08-15/proxy/tianba-kitchen-service/tianba-kitchen-api/
   ```
3. 复制这个地址！

---

## 第五步：测试 API

在浏览器新标签页打开：
```
https://你的API地址/api/auth/check
```

如果看到类似这样的返回：
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "verified": false
  }
}
```

说明部署成功！🎉
